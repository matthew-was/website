At the time programming as a career was never made obvious to me otherwise things might be very different today.During my time at University as an undergraduate I became aware of more programming languages and was first introduced to Fortran77 and Fortran 95. It was during my doctorate that my interest in programming was rekindled with basic knowledge of Bash, IgorPro, SPEC and Python.

During my postdoctoral work, still based in science, I learnt more about Bash and Python, in the form of SASView and was introduced to Java and C, although only in passing including discussions on adapting the fitting routines in SASView to C and CUDA to allow data fitting using GPGPU. First job as a developer

Since starting my first position as a junior developer I now have experience with a range of different technologies. I have worked with client and server side Javascript in docker containers deployed in a mesos cluster. While I have not been directly involved in the infrastructure, I have completed small projects that have exposed me to the AWS platform and Ansible as a deployment techique, and the use of Githubs Hubot.

While the majority of new code was being written in Javascript I also spent time working with the legacy system where the main codebase was in C#, a mixture of .NET web applications and console applications on both IIS6 and IIS8 servers. Also part of this system was a Java app, run as a dockerised container in the mesos cluster. A major part of these systems that I had not previously been exposed to was working with databases, in this case MS-SQL and Postgres.
